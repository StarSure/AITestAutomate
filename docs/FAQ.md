# 常见问题

## 1. 这个项目现在能做什么？

当前版本可以：

- 直接输入网页地址进行抓取
- 自动采集页面元素
- 导入 OpenAPI / Swagger
- 导入 Postman Collection
- 导入 HAR 文件
- 导入 cURL
- 导入请求 JSON 样本
- 自动识别接口
- 自动生成接口测试用例
- 执行测试并展示报告

## 2. 现在支持网页录制吗？

已经支持基础网页抓取能力。

当前版本可以直接输入网页地址，自动尝试登录，并采集页面元素和请求。后续会继续增强更稳定的网页录制和登录后接口抓取。

## 3. 现在适合直接用于生产环境吗？

不建议。

当前版本更适合：

- 本地环境
- 测试环境
- 预发环境
- Demo 环境

## 4. 现在支持哪些导入方式？

当前支持：

- OpenAPI / Swagger
- Postman Collection
- HAR
- cURL
- 手动 JSON 请求样本

## 5. 这个项目和普通 API 测试工具有什么不同？

区别在于它不是单纯的接口调试工具，而是希望把“网页业务流量 -> 接口资产 -> 测试用例 -> 执行报告”串成一个完整工作流。

## 6. 后续会支持什么？

后续计划支持：

- Apifox 导入
- 更稳定的 Playwright 网页录制
- 登录后业务接口深度采集
- 测试历史记录
- SQLite / PostgreSQL
- GitHub 集成
- Docker Compose 部署
